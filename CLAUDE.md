# PLATFORMANCE — память проекта для Claude

Статический лендинг PLATFORMANCE (системное ведение магазинов на Wildberries/Ozon).
Чистый HTML/CSS/JS без сборщика, хостится на GitHub Pages.

- Репозиторий: `alexblacklucky/PLATFORMANCE`
- Прод-ветка: `main`
- Pages URL: `https://alexblacklucky.github.io/PLATFORMANCE/`
- Canonical-домен в метатегах/JSON-LD: `https://uhjr7sv23.urest.org/` (старый, требует ревизии при смене домена)
- Оператор/владелец: ИП Чалова Татьяна Владимировна

## Структура

- `index.html` — главная, форма заявки `.lead-form` (`data-lead-form`)
- `privacy.html` — политика конфиденциальности (152-ФЗ)
- `thank-you.html`, `404.html` — служебные страницы (app.js НЕ подключён)
- `assets/app.js` — меню, табы, модалки, валидация формы, cookie-баннер
- `assets/config.js` — `window.PLATFORMANCE_CONFIG={webhookUrl,thankYouPage}` + список `files=[...]` для динамической подгрузки `v2-*.css/js`
- `assets/styles.css` — базовые стили (синхронная загрузка, есть в `<head>`)
- `assets/v2-*.css`, `assets/v2-dom-*.js`, `assets/v2-patches*.js` — слой визуального рефайна "v2", подгружается **асинхронно** через `config.js` после первого рендера

### ⚠️ Важная архитектурная ловушка v2-слоя
`config.js` хранит явный список файлов для динамической подгрузки. **`assets/v2-dom-1-4.js` в этот список НЕ включён** — он мёртвый код, никогда не выполняется. Раньше казалось, что он отвечает за блок «Система», но реальная разметка этого блока — в `index.html` напрямую, а `v2-patches-more.js` дополняет её в рантайме (кольца, theme-классы). **Перед правкой любого визуального бага сначала проверяй, действительно ли файл, который собираешься редактировать, загружается** (сверяйся со списком `files` в `config.js`), иначе правки уйдут в никуда.

6. **Исправление радар-анимации блока «Система»**:
   - Баг: подпись-тег «Точка заказа» проваливалась под карточку «Реклама» — плавающие `.system-floater` имели фиксированные px-координаты без `z-index`, накладывались на `.system-node` (z-index:4) при некоторых ширинах экрана.
   - Решение (по выбору пользователя — концепция «радар/сонар-пульс»): узлы переставлены в кардинальные точки (верх/право/низ/лево) вокруг ядра; подпись-тег теперь `<small class="node-tag">` **внутри** кнопки-узла, а не отдельный плавающий элемент — наложение исключено структурно. Добавлены расходящиеся импульсные кольца (`.radar-pulse`) и вращающийся луч развёртки (`.radar-sweep`), пульсация ядра. `prefers-reduced-motion` отключает анимации.
   - Правки внесены в `index.html` (реально используемая разметка), `assets/styles.css` (база + fallback-позиционирование на случай сбоя загрузки v2-слоя), `assets/v2-0.css` (доп. стили радара и мобильные брейкпоинты), `assets/v2-patches-more.js` (убрана мёртвая логика создания floater-тегов).
   - Попутно найден и исправлен независимый баг: на ширине 651–880px `.system-map` схлопывался до ~46px из-за `margin:auto` без явного `width` в паре с absolute-позиционированными детьми (затрагивало весь сайт, не только новый дизайн).
   - Проверено Playwright на ширинах 1920/1024/768/480/390/360px — наложений нет, клики по узлам переключают карточку деталей корректно.
   - `assets/v2-dom-1-4.js` тоже был отредактирован для консистентности, но он не используется (см. ловушку выше) — это не баг, осознанное решение не тратить время на синхронизацию мёртвого кода.
